PortableApps.com (Evernote Sticky Portable) セットアップおよび利用方法

事前準備
作業ディレクトリと作成に利用するアプリのディレクトリを用意
アプリはそちらにインストールする
ex.
PAWork\EvernoteStickyPortable
PATools\PortableApps.comInstaller
PATools\NSISPortable
といった具合

ランチャー作成
利用するPortableApps.comのアプリ
  • PortableApps.com Launcher
    • NSIS (Nullsoft Scriptable Install System) Portable (Unicode)
が必要
 
フォルダ構成など
AppNamePortable
+ App
  + AppInfo
    + FileTypeIcons
  + AppName
  + DefaultData
+ Data
+ Other
  + Help
    + Images
  + Source

「AppName」は「EvernoteStickyPortable」に置き換えて読むこと

ここで利用するiniファイル
\App\AppInfo\EvernoteStickyPortable.ini (ランチャー用)
\App\appinfo.ini (アプリ用)
\App\installer.ini (インストーラ用追加設定)

実行ファイル
\App\EvernoteStickyPortable配下に設置

データファイル
\App\DefaultDataフォルダにSticky.ini、icon.pngを入れること
# iniにワーキングディレクトリとしてJava VMを起動する設定にしているため、ここが設定、icon.png、cacheやlockなどの生成の対象となる
また、icon.pngからWindowsアイコンファイルを生成しているので、\Appに配置

\help.htmlを用意する

実行
上記状態でLauncherアプリを実行する。
ディレクトリを聞いてくる
ので、EvernoteStickyPortableディレクトリを指定する

と
このようにランチャーアプリが生成される。

インストーラにパッケージング
利用するPortableApps.comのアプリ
  • PortableApps.com Installer
が必要

実行
上記状態でInstallerアプリを実行する。
ディレクトリを聞いてくる
のでEvernoteStcikyPortableディレクトリを指定する

その後、自動でPAWork(PAToolかも)の直下にインストーラパッケージが生成される

実行環境
利用するPortableApps.comのアプリ
  • jPortable
が必要

実行
jPortableとEvernoteStickyPortableを同じアプリディレクトリにインストールして
EvernoteStickyPortable.exeを起動するとJVMの起動経由でEvernote Stickyが起動する。

カスタマイズ
\Dataに\App\DefaultDataがコピーされるので、実行環境のカスタマイズ(Sticky.iniの編集)はそちらについて行う。
